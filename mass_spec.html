def read_ms_mxXML(file):                                                                                                                                                                                        
    exp = oms.MSExperiment()                                                                                                                                                                                    
    oms.MzXMLFile().load(file, exp)                                                                                                                                                          
    return exp.getSpectrum(0).get_peaks() # Spectrum::([mz], [intensity])

It would be nice to be able to group these clusters of peaks. In general, clustering can be a hard problem, but here, we know that the members in each group should be no more than 5 or 6 apart from each other.

I started writting code, but realized that someone had already solved this problem. This function is not mine.

In [153]:
def group_by_distance(arr, dist):
    # Sort the array along with their original indices
    sorted_arr_with_indices = sorted(enumerate(arr), key=lambda x: x[1])
    
    # Initialize the first group
    groups = [[sorted_arr_with_indices[0][0]]]
    
    # Iterate and group elements
    for idx, element in sorted_arr_with_indices[1:]:
        # Check if the element is close enough to the last element of the last group
        if element - arr[groups[-1][-1]] <= dist:
            groups[-1].append(idx)
        else:
            # Start a new group
            groups.append([idx])
    
    return groups

That looks pretty good! Each of the groups corresponds to a cluster of peaks. This way we can compare the max of two groups and begin to quantify things like efficiency.

We can also pick out the principal member of a group and use it to measure distance between the principal member of other groups. We can potentially weed out K and Na signals and flag unwanted side products.

We can look at the relative heights of the peaks within a grouping to see if there are problems.

def cluster_dist(group1, group2, idx=group_idx):
    group1, group2 = group1 - 1 , group2 - 1 # Python is 0-based
    return round(abs(mz[group_idx[group1]][0] -  mz[group_idx[group2]][0]),2)
